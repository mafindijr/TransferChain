# Contract ABIs

## What Is This Folder?

The `abis/` directory contains clean, production-ready ABI JSON files for all 8 deployed TransferChain smart contracts. Each file contains **only** the ABI array — no bytecode, metadata, storage layouts, or compiler artifacts.

## Why Does It Exist?

The official `@transferchain/sdk` npm package cannot be republished at this time due to an npm OTP issue. The frontend team needs the contract ABIs immediately to continue development.

This folder serves as a **temporary integration layer** until the SDK package is republished. Once the npm issue is resolved, the SDK will include these ABIs directly, and this folder can be removed.

## Contract ABIs

| File | Contract | Description |
|------|----------|-------------|
| `TransferChainAccessControl.json` | TransferChainAccessControl | Roles, pause/unpause |
| `TransferChainConfig.json` | TransferChainConfig | Treasury, fees, tokens, emergency mode |
| `PlayerRegistry.json` | PlayerRegistry | Player registration, metadata, status |
| `ClubRegistry.json` | ClubRegistry | Club registration, metadata, status |
| `TransferMarketplace.json` | TransferMarketplace | Listings and offers |
| `TransferAgreementManager.json` | TransferAgreementManager | Transfer agreements |
| `Escrow.json` | Escrow | ERC-20 escrow deposits |
| `Treasury.json` | Treasury | Protocol treasury |

## How to Import

### ethers.js v6

```ts
import PlayerRegistryABI from "../abis/PlayerRegistry.json";

const contract = new ethers.Contract(address, PlayerRegistryABI, provider);
```

### viem

```ts
import PlayerRegistryABI from "../abis/PlayerRegistry.json";

const client = createPublicClient({ ... });
const data = await client.readContract({
  address,
  abi: PlayerRegistryABI,
  functionName: "getPlayer",
  args: [ownerAddress],
});
```

### React / Next.js

```ts
// Direct import
import TransferChainAccessControlABI from "../../abis/TransferChainAccessControl.json";

// Or fetch at runtime
const response = await fetch("/abis/TransferChainAccessControl.json");
const abi = await response.json();
```

### Dynamic Import (tree-shakeable)

```ts
const EscrowABI = (await import("../abis/Escrow.json")).default;
```

## File Format

Each file is a JSON array of ABI entries:

```json
[
  {
    "type": "function",
    "name": "getPlayer",
    "inputs": [
      { "name": "owner", "type": "address" }
    ],
    "outputs": [
      {
        "name": "",
        "type": "tuple",
        "components": [
          { "name": "id", "type": "uint256" },
          { "name": "owner", "type": "address" },
          { "name": "name", "type": "string" }
        ]
      }
    ],
    "stateMutability": "view"
  }
]
```

## When Will This Be Removed?

Once the `@transferchain/sdk` npm package is republished with the updated ABIs, this folder will be removed. The SDK will export all ABIs as typed constants:

```ts
import { PLAYER_REGISTRY_ABI, TransferChain } from "@transferchain/sdk";
```

Until then, use the files in this folder directly.

## Source

These ABIs were exported from the Foundry compiled artifacts (`out/`) generated by `forge build` against the deployed Solidity contracts. They match the deployed contracts on Injective Testnet (chain ID 1439).
